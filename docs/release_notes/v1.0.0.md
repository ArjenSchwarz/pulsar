# v1.0.0 — 2026-05-18

Initial public release of Pulsar.

Pulsar is a small Go binary that turns a directory of locally-generated code review HTML files into an [RSS 2.0 feed](https://www.rssboard.org/rss-specification), designed to be served over [Tailscale](https://tailscale.com/) so you can subscribe to your own reviews from any device on your tailnet.

## What's in it

- **`pulsar publish <file>`** — validates a review HTML file (checks the embedded JSON metadata block), normalises the repo URL, stamps the timestamp, canonicalises the filename, drops it into `<archive>/YYYY-MM/`, and removes any prior version for the same PR or branch. Writes are atomic (temp file + rename).
- **`pulsar serve`** — long-running HTTP server bound to `127.0.0.1` that exposes the archive as static files, a minimal HTML index at `/`, and an RSS 2.0 feed at `/feed.xml`. Front it with `tailscale serve` (or `tailscale funnel` for public access) — see the README for the one-line command.
- **`pulsar install`** — generates and loads a macOS LaunchAgent (`io.arjen.pulsar`) that runs `serve` on login. Auto-detects a Tailscale `*.ts.net` hostname when `--base-url` isn't supplied.
- **`pulsar uninstall`** — unloads and removes the LaunchAgent.

## Configuration

Resolution order (highest priority first):

1. Command-line flags
2. Environment variables: `PULSAR_DIR`, `PULSAR_PORT`, `PULSAR_BASE_URL`, `PULSAR_TITLE`, `PULSAR_DESCRIPTION`, `PULSAR_MAX_ITEMS`
3. `$HOME/.config/pulsar/config.yaml`
4. Built-in defaults (archive at `$HOME/CodeReviews`, port `8765`, max 200 feed items)

`--base-url` is required for `serve` (it's used to build absolute links in the feed).

## Installation

Pre-built binaries for `linux`, `darwin`, and `windows` on `amd64` and `arm64` are attached to this release. Or:

```sh
go install github.com/ArjenSchwarz/pulsar@latest
```

The `install` / `uninstall` subcommands are macOS-only (they call `launchctl`). The rest of the binary runs anywhere Go does.

## Agent contract

The metadata schema and validation rules consumed by review-generating agents are documented in [`docs/agent-contract.md`](../agent-contract.md). At minimum, every review HTML file must contain a single `<script type="application/json" id="review-meta">` block with `title`, `repoUrl`, `severity` (`lgtm` / `suggestions` / `needs-changes` / `blocking`), `summary`, and exactly one of `pr` or `branch`.
